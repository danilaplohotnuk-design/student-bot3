# Екзамени в Supabase (щоб не зникали після деплою)

У коді вже все є: якщо на сервері є `SUPABASE_URL` і `SUPABASE_SERVICE_ROLE_KEY`, екзамени пишуться в таблицю **`schedule_exams`**.

## Що зробити тобі (один раз)

### 1. Таблиця в Supabase

1. Зайди в [Supabase](https://supabase.com) → свій проєкт.
2. **SQL Editor** → **New query**.
3. Скопіюй **увесь** вміст файлу **`exams.sql`** з цієї папки і натисни **Run**.
4. Має виконатися без помилок (таблиця `schedule_exams` з’явиться в **Table Editor**).

> Якщо «Важливо» вже на Supabase — `SUPABASE_URL` у Render уже є. Додаткових ключів не треба.

### 2. Змінні на сервері (Render тощо)

- **`SUPABASE_URL`** — Project URL (Settings → API).
- **`SUPABASE_SERVICE_ROLE_KEY`** — **service_role** (secret), **не** anon key.

Після зміни env — **Redeploy** сервісу.

### 3. Перевірка

У логах після старту має бути рядок:

`Екзамени: Supabase (таблиця schedule_exams)`

Якщо бачиш `Екзамени: файл exams.json` — змінні не підхопились або сервер не перезапущено.

---

**Міграція зі старого `exams.json`:** автоматично не переноситься. Старі записи можна додати вручну в адмінці або один раз імпортувати в таблицю (за бажанням — окремо).
